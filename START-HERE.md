# START-HERE — твой первый Telegram-бот за час

> **Зачем это нужно.** Это короткий вход. Ты создашь пустую папку, откроешь в ней Claude Code, вставишь один готовый промпт — и Claude сам скачает учебный пакет и проведёт тебя **за руку** до работающего бота. Здесь только старт; каждый следующий шаг Claude объяснит сам.

---

## ✅ Что подготовить заранее (10–30 минут)

Пройди по списку. Чего-то нет — **доделай до старта**, иначе застрянешь на середине.

- [ ] **Claude Code** — активная **платная подписка** (Pro/Max или Console) и ты умеешь им пользоваться (CLI или приложение). Бесплатный чат claude.ai **не подходит**.
- [ ] **Node.js (LTS)** установлен — на нём поднимается «мостик» Claude ↔ n8n. Скачай на [nodejs.org](https://nodejs.org) (кнопка **LTS**), установи. **Без него бот не соберётся** — это был самый частый затык на живом прогоне, поставь заранее (5 минут).
- [ ] **Свой n8n.** Проще всего — облачный **n8n Cloud** (есть бесплатный пробный период): регистрируешься сам и сразу получаешь публичный адрес. Можно и self-hosted **Community Edition**, но он должен быть **доступен из интернета по HTTPS** — Telegram шлёт сообщения через webhook, локальный n8n их не примет. Подробнее — раздел «Где взять n8n».
- [ ] **API-ключ n8n** — в твоём n8n: **Settings → n8n API → Create API key**. Это секретная строка доступа (не пароль от аккаунта).
- [ ] **Адрес твоего n8n** под рукой (напр. `https://твой-субдомен.app.n8n.cloud`) — **без `/` в конце**.
- [ ] **Токен Telegram-бота** — получен у **@BotFather** (встроенный сервис Telegram для создания ботов).
- [ ] **Username в Telegram** задан (`@имя`) — по нему бот проверяет доступ. Нет — задай в настройках Telegram.
- [ ] **Ключ AssemblyAI** — сервис распознавания речи. Зарегистрируйся на [assemblyai.com](https://www.assemblyai.com) ($50 бесплатного кредита, без карты); ключ — на главной странице дашборда после входа. **Запомни свой регион** (EU или US) — понадобится при настройке бота.

> **Если Claude Code выдаёт ошибку аутентификации или «не могу подключиться»** — проверь, что сервисы Anthropic доступны из твоего региона (в ряде стран нужен VPN).

> Пакет рассчитан и протестирован на **Claude Code**. Другой агентный инструмент с поддержкой MCP (напр. Codex CLI) в принципе может подойти, но стартовый промпт и `.mcp.json` заточены под Claude Code — **совместимость с аналогами не гарантируется**.

## Где взять n8n

- **Облако (рекомендуется).** Зайди на [n8n.io](https://n8n.io) → начни пробный период → заведи аккаунт. Получишь готовый публичный адрес вида `https://твой-субдомен.app.n8n.cloud`: Telegram-webhook работает сразу, VPN и настройка сервера не нужны.
- **Self-hosted (Community Edition).** Если разворачиваешь сам — n8n должен быть доступен **из интернета по HTTPS** (свой домен или туннель). Локальный `localhost` для Telegram-бота не годится.

## Шаг 1. Создай папку и открой в ней Claude Code

- Создай новую **пустую папку** для проекта — там, где обычно держишь проекты Claude Code. Назови как удобно, напр. `bot-workshop`.
- Открой **Claude Code в этой папке**.

Больше ничего вручную качать не нужно: **пакет заберёт сам Claude** по промпту из Шага 2.

> Git и SSH-ключи тебе **не нужны** — Claude скачает пакет сам (архивом, если git не установлен).

## Шаг 2. Вставь стартовый промпт

Скопируй этот текст **целиком** и отправь Claude **первым сообщением**:

```
Ты — Claude Code в РЕЖИМЕ МАСТЕР-КЛАССА (учебный воркшоп, ~1 час).

Папка сейчас пустая. Сначала забери в неё учебный пакет из репозитория:
https://github.com/sergeykruglovit-a11y/bot-starter-workshop
Скачай его содержимое в ТЕКУЩУЮ папку любым надёжным способом (git не
обязателен — если его нет, скачай и распакуй архив репозитория). Убедись,
что рядом появились файлы CLAUDE-workshop.md и START-HERE.md.

Объяви режим мастер-класса и назови задачу (Telegram-бот: голосовое →
распознавание AssemblyAI → текст ответом, только разрешённым по списку).

Затем СНАЧАЛА помоги подключить n8n:
- скопируй .mcp.json.example в .mcp.json;
- покажи, какие ДВЕ строки заменить: адрес моего n8n (N8N_API_URL, БЕЗ «/»
  в конце) и ключ n8n (N8N_API_KEY); менять только значения, кавычки не трогать;
- напомни, что ключ — секрет: я впишу его в файл САМ, в чат не присылаю;
- попроси меня ПОЛНОСТЬЮ закрыть и заново открыть Claude Code (конфиг MCP
  читается при старте) и дай короткую фразу-возврат, с которой я вернусь и
  ты продолжишь по CLAUDE-workshop.md с Шага 0.

Дальше веди строго по CLAUDE-workshop.md: один шаг за раз (жди моего «готово»),
workflow создавай active:false (активирую и тестирую я сам), секреты в чат не
проси. Начни с приветствия и подключения пакета.
```

## Шаг 3. Впиши доступ к n8n и перезапусти Claude Code

Claude создаст файл **`.mcp.json`** и покажет, что в нём править. Тебе нужно:

- Вписать **адрес своего n8n** (напр. `https://твой-субдомен.app.n8n.cloud`) — **без `/` в конце** (лишний слэш = «нет связи»).
- Вписать **свой API-ключ n8n**.
- Менять **только значения** в кавычках (адрес и ключ). Сами кавычки и остальные строки не трогай.
- **Полностью закрыть и заново открыть Claude Code** — конфиг подключения читается только при старте.

> `.mcp.json` — скрытый файл (имя с точкой). Его создаёт Claude, тебе нужно лишь вписать две строки в редакторе. Ключ остаётся в файле на твоём компьютере и **не уходит в чат**.

После перезапуска вставь **фразу-возврат** (её даст сам Claude), например:

```
Продолжаем мастер-класс. Прочитай CLAUDE-workshop.md и начни с Шага 0 —
.mcp.json уже настроен.
```

> Первое подключение может занять ~минуту: `n8n-mcp` один раз скачивает базу описаний нод. Это нормально — дождись.

## Шаг 4. Дальше — следуй за Claude

Claude проверит связь и поведёт по шагам. **Твоя роль:** делать по одному действию и писать «готово» — тогда идёт следующий шаг. Не торопись вперёд.

> Совет: для этой задачи достаточно модели **Claude Sonnet** — она справляется, а расход меньше. Opus здесь избыточен.

## 🔑 Куда пойдут твои ключи (и почему не в чат)

| Ключ | Куда вводишь | Кто вводит |
|------|--------------|:----------:|
| API-ключ **n8n** | в файл `.mcp.json` (на твоём компьютере) | ты сам |
| Токен **Telegram** | в **credentials n8n** (в интерфейсе n8n) | ты сам |
| Ключ **AssemblyAI** | в **credentials n8n** | ты сам |

**Никогда не отправляй ключи в переписку с Claude.** Всё, что попадает в чат, уходит в модель. Claude о ключах в чат не просит — он только подсказывает, куда их вписать.

## Что мы построим

Голосовое в Telegram → проверка доступа по списку → распознавание в AssemblyAI → текст ответным сообщением.

Маршрут часа: **связь с n8n → ключи в n8n → таблица доступа (твой `@username`) → бот собирается → активация → тест голосовым**.

## Не успел за час?

Ничего страшного — **пакет остаётся у тебя**. Файл `CLAUDE-workshop.md` рядом — это полная инструкция; продолжишь с любого места, и Claude так же поведёт тебя за руку.
