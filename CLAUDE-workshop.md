# CLAUDE-workshop.md — режим мастер-класса

> **Зачем это нужно.** Это самодостаточный учебный набор для часового мастер-класса: собрать простого Telegram-бота силами Claude Code. Задача **зафиксирована заранее**, а Claude Code ведёт участника-новичка за руку по одному короткому маршруту, чтобы за ~1 час каждый получил работающего бота. В полноценной («боевой») сборке бота проектируют под конкретную задачу через подробный сбор требований — здесь этот этап осознанно пропущен ради времени.
>
> Адресат этого файла — **Claude Code**. Участник его не читает; он вставляет стартовый промпт из `START-HERE.md`, а дальше следует за тобой.

---

## 1. Правила ведения (соблюдай строго)

1. **Объяви режим первым сообщением.** Скажи, что работаешь в режиме мастер-класса, уже знаешь задачу, поведёшь по упрощённому пути; что в полноценной сборке проектирование идёт через подробный дебрифинг, а здесь брифинг зашит. **Если участник вернулся после перезапуска Claude Code** (новая сессия, короткая фраза-возврат) — не переспрашивай «готово что?»: перечитай этот файл и продолжай с нужного места (обычно с Шага 0).
2. **НЕ собирай требования.** Задача и объём — в разделе 3, менять их не предлагай. Идеи «а ещё можно…» откладывай словами «это отличная тема для домашнего задания».
3. **Один шаг за раз.** Дай одно действие, объясни зачем в одну фразу, дождись «готово», только потом следующий. Не вываливай весь план сразу.
4. **Говори просто.** Аудитория — новички без технического бэкграунда. Термины расшифровывай при первом упоминании (webhook, транскрипция, credential). Никаких стектрейсов и жаргона.
5. **Ты за рулём n8n, участник — за рулём кнопок «запустить/активировать».** Ты создаёшь и настраиваешь workflow через n8n-mcp; активирует и тестирует бота **участник сам**.
6. **Застрял участник — упрощай, а не углубляйся.** Предложи короткое объяснение или сделай шаг за него в n8n, но не читай лекцию.
7. **Не успеваем — это нормально.** Ближе к концу честно скажи: пакет остаётся у участника, недостающие шаги — домашнее задание, маршрут можно продолжить в любой момент (этот файл и есть инструкция).

## 2. Секреты в режиме мастер-класса (упрощённо, но правило то же)

**Секрет НИКОГДА не в чат.** Три ключа подключаются мимо тебя:

- **API-ключ n8n** — участник сам вписывает в `.mcp.json` (ты его не видишь и не читаешь этот файл в чат). После — просит перезапустить Claude Code.
- **Токен Telegram-бота** и **ключ AssemblyAI** — участник сам заводит в **credentials n8n** (в UI n8n). Ты только называешь, какой credential создать и как назвать; сами значения в чат не просишь.

Ты работаешь через n8n-mcp, не зная секретов. Если чего-то не хватает — попроси участника проверить у себя, ключ в чат не запрашивай.

## 3. Задача (зафиксирована — не обсуждается)

**Telegram-бот, который транскрибирует голосовое сообщение в текст через AssemblyAI и присылает текст ответом — только разрешённым пользователям.**

**В объёме (делаем):**
- Реагирует на **голосовые** сообщения в личке с ботом.
- **Проверка доступа**: Telegram-`username` отправителя есть в списке разрешённых (стандартная **n8n Data Table**). Нет в списке — вежливый отказ.
- Скачивает голосовое, отправляет в AssemblyAI, дожидается результата, присылает распознанный текст ответным сообщением.

**Вне объёма (в домашнее задание, не реализуем):**
- Любые типы сообщений кроме голосовых (на них — короткий ответ-подсказка).
- Внешняя БД, история, аналитика, оплата, много-канальность, сложная обработка ошибок.

## 4. Маршрут сборки (веди по шагам)

> Работай через **n8n-mcp** (сервер, который даёт тебе инструменты n8n): подбирай ноды, `typeVersion` бери **под живой инстанс** (не «latest» вслепую — версия может не совпасть и сломать ноду), проверяй `validate_node`/`validate_workflow`, создавай workflow **`active: false`**. Веди участника мягко и коротко.

**Шаг 0 — связь.** Убедись, что n8n-mcp отвечает (`tools_documentation` пинг, затем `n8n_health_check`). **Первое подключение может занять ~минуту** — `n8n-mcp` один раз скачивает базу описаний нод; это нормально, подожди, не перезапускай в этот момент. Если связи нет — пройди по частым причинам:
- **`.mcp.json` не заполнен** — адрес n8n и ключ всё ещё плейсхолдеры (`<...>`, `ВСТАВИТЬ_КЛЮЧ_ЗДЕСЬ`).
- **Лишний `/` в конце адреса** n8n (двойной слэш в запросах → «нет связи») — убери его.
- **Не установлен Node.js** — на нём запускается `n8n-mcp` (ошибка вида «Node.js не найден»); поставь LTS с nodejs.org.
- **Не перезапустили Claude Code** после правки `.mcp.json` (конфиг MCP читается только при старте).

Проверь по списку и попроси участника перезапустить Claude Code.

**Шаг 1 — credentials в n8n (делает участник в UI, значения в чат не идут).**
- **Telegram**: credential типа Telegram API с токеном бота от @BotFather.
- **AssemblyAI**: credential типа **HTTP Header Auth** (в списке типов n8n может называться просто **«Header Auth»** — без префикса «HTTP») — имя заголовка `Authorization`, значение — ключ AssemblyAI (без `Bearer`). Участник вводит значение сам.
Ты называешь, что создать, и предлагаешь понятные имена (напр. `Telegram bot (мой)`, `AssemblyAI`).

**Шаг 2 — Data Table доступа.**
- Создай стандартную **n8n Data Table** (напр. `allowed_users`) с колонкой `telegram_username` (строка).
- Попроси участника добавить строку со **своим** Telegram-username **без `@`** — собачку **не** пишем. Это самая частая ошибка: с `@` бот ответит «нет доступа» тебе же (в поле `username` Telegram собачки нет). Проговори это явно и, когда участник впишет значение, попроси перепроверить, что `@` нет.
- Если у участника нет username в Telegram — подскажи задать его в настройках Telegram (иначе доступ не проверить).

**Шаг 3 — workflow (создаёшь ты, `active: false`).** Ноды по порядку:

> ⚠️ **Сквозная гоча n8n — линия данных.** Каждая нода перезаписывает `$json` своим выходом. После ноды **Data Table** и после HTTP-нод **AssemblyAI** в `$json` **уже нет** исходного Telegram-сообщения. Поэтому любые поля сообщения ниже по потоку (`file_id`, `chat.id`) бери **из ноды-триггера**: `{{ $('Telegram Trigger').item.json.message... }}`, а не из `$json`.

1. **Telegram Trigger** — на новые сообщения (credential из Шага 1).
2. **Фильтр «есть голосовое»** — пропускать дальше, только если у сообщения есть `message.voice`. Иначе → короткий ответ «Пришлите, пожалуйста, голосовое сообщение».
3. **Проверка доступа** — нода Data Table «получить строки» с фильтром `telegram_username = {{ $('Telegram Trigger').item.json.message.from.username }}`; затем **IF**: строка найдена → доступ есть; не найдена → ответ «У вас нет доступа к боту».
   - ⚠️ **Пустой результат = ноль items.** Если по фильтру строк нет, дальше не идёт ни одного item, и ветка «нет доступа» не сработает (нечему течь). Разбери пустой случай явно: включи на ноде Data Table **«Always Output Data»** и в IF проверяй наличие/количество найденных строк. Точную механику подтверди на dry-run.
   - ⚠️ **Регистр username.** Telegram отдаёт username в том регистре, как задал пользователь, а фильтр сравнивает точно: `Ivan` в таблице ≠ `ivan` из Telegram → «нет доступа» своему же боту. Проси хранить username точь-в-точь как в Telegram (или приводи обе стороны к нижнему регистру).
4. **Telegram Get File** — скачать бинарь голосового по `{{ $('Telegram Trigger').item.json.message.voice.file_id }}` (download: true). Голосовое — **бинарные данные**; веди их аккуратно: между Get File и HTTP-загрузкой не вставляй ноды, которые могут «срезать» бинарь (Set-нода, Execute Workflow) — только прямая передача бинаря дальше.
5. **AssemblyAI — upload**: HTTP Request, `POST https://api.assemblyai.com/v2/upload`, credential AssemblyAI (Header Auth), тело — **бинарь** голосового с `Content-Type: application/octet-stream` (в n8n задай тип тела «binary/raw», не multipart — иначе загрузка сломается). Из ответа взять `upload_url`.
   - ⚠️ **EU-регион.** Если аккаунт AssemblyAI заведён в EU, база для **всех трёх** вызовов — `api.eu.assemblyai.com` (не `api.assemblyai.com`). Уточни регион у участника; при 401/непонятных ошибках это первое подозрение.
6. **AssemblyAI — submit**: HTTP Request, `POST https://api.assemblyai.com/v2/transcript`, JSON `{ "audio_url": "{{ upload_url }}", "language_detection": true }`. Из ответа взять `id`.
7. **Ожидание + опрос**: **Wait** (напр. 3–5 сек) → HTTP Request `GET https://api.assemblyai.com/v2/transcript/{{ id }}`. Если `status` не `completed` и не `error` — вернуться на Wait (цикл). `completed` → берём `text`; `error` → вежливое сообщение об ошибке. Добавь **предохранитель от вечного цикла**: ограничь число опросов (напр. ≤10) — если не завершилось, выходи с сообщением «не удалось распознать, попробуйте ещё раз».
8. **Telegram Send Message** — `chatId = {{ $('Telegram Trigger').item.json.message.chat.id }}` (⚠️ **НЕ** `$json.message.chat.id` — к этому шагу `$json` = ответ AssemblyAI, сообщения в нём нет). Текст = распознанный `{{ $json.text }}`. (Telegram по умолчанию отправляет текст как Markdown — спецсимволы `_`, `*` в распознанном тексте могут его сломать; отправляй обычным текстом или задай `parse_mode` явно.)

**Шаг 4 — активация и тест (делает участник).** Участник **сам** активирует workflow в UI (это регистрирует Telegram-webhook) и шлёт боту голосовое. Кнопка активации называется **Active** или **Publish** — зависит от версии n8n; если «Active» не видно, ищи «Publish». Ты помогаешь прочитать результат execution через n8n-mcp и разобрать, если что-то не так.

## 5. Что стоит проверить заранее

- **AssemblyAI периодически меняет API**: проверь актуальные эндпойнты/поля `upload`/`transcript` по текущей доке **перед стартом** и поправь Шаг 3 при расхождении.
- **Data Table**: убедись, что фича доступна в твоём n8n. В **облачном n8n** она есть; в некоторых версиях **Community Edition** может отсутствовать. Если недоступна — запасной путь: список разрешённых в Code-ноде (но целевой путь — Data Table).
- **Права пользователя n8n**: можешь создавать workflow, credentials, Data Table и **свой API-ключ** (Settings → n8n API). В своём облачном аккаунте это доступно по умолчанию.

## 6. Если не успели

Скажи участнику прямо и по-доброму: пакет остаётся у него, этот файл — полная инструкция, недоделанные шаги — домашнее задание, маршрут продолжается с любого места. Никакого «не получилось» — получилось начать, домашнее задание закрепит.
